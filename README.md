## 🎯 Overview

**MatruRaksha** (Mother's Protection) is a comprehensive maternal healthcare platform designed specifically for rural and underserved areas of India. The system bridges the gap between pregnant mothers, ASHA workers (frontline health workers), and doctors through intelligent AI-powered monitoring and real-time communication.

### The Problem We Solve

Rural India faces critical maternal healthcare challenges:
- ❌ Limited access to qualified doctors
- ❌ Delayed detection of pregnancy complications
- ❌ Communication barriers between healthcare workers and facilities
- ❌ Lack of medical history tracking
- ❌ Language barriers in accessing medical knowledge
- ❌ No real-time health monitoring

### Our Solution

MatruRaksha provides:
- ✅ **AI-powered risk assessment** for early complication detection
- ✅ **RAG-based chatbot** for ASHA workers with medical knowledge
- ✅ **Doctor AI Assistant** for clinical decision support
- ✅ **Telegram bot** for mother self-registration and communication
- ✅ **Real-time health monitoring** through ASHA workers
- ✅ **Comprehensive dashboards** for all stakeholders
- ✅ **Document management** for medical records
- ✅ **Multi-language support** (planned)

## 🤖 AI Models & Components

### 1. **Llama 3.3 70B Versatile** (via Groq)
**Used For:**
- ✅ Doctor AI Assistant case analysis
- ✅ Complex medical reasoning
- ✅ Comprehensive assessment comparisons
- ✅ Detailed trend analysis
- ✅ Critical decision support

**Why 70B?**
- Superior reasoning capability
- Better medical knowledge
- More accurate trend detection
- Nuanced understanding of complex cases

**Configuration:**
```python
model = "llama-3.3-70b-versatile"
temperature = 0.1  # Low for factual medical analysis
max_tokens = 1500
```

### 2. **Llama 3.1 8B Instant** (via Groq)
**Used For:**
- ✅ ASHA RAG chatbot responses
- ✅ Nutrition advisor recommendations
- ✅ Quick risk assessments
- ✅ Real-time chat interactions
- ✅ Conversational AI

**Why 8B Instant?**
- Fast response time (<1 second)
- Good for conversational tasks
- Cost-effective for high-volume queries
- Sufficient for guided responses (RAG context)

**Configuration:**
```python
model = "llama-3.1-8b-instant"
temperature = 0.3  # Balanced for conversation
max_tokens = 500-800
```

### 3. **LangChain RAG Pipeline**
**Components:**
- **Document Loader**: Loads ASHA manuals (PDF)
- **Text Splitter**: Chunks documents (500 chars, 50 overlap)
- **Embeddings**: Sentence Transformers (all-MiniLM-L6-v2)
- **Vector Store**: ChromaDB with semantic search
- **Retriever**: Top-k similarity search (k=4)
- **LLM**: Groq Llama 3.1 8B
- **Prompt Template**: Structured medical Q&A format

**Flow:**
```
User Query → Embeddings → Vector Search → 
Top 4 Chunks → Groq LLM → Formatted Response + Sources
```

### 4. **LangSmith Integration**
**Features:**
- ✅ **Tracing**: Track all LLM calls, latency, token usage
- ✅ **Debugging**: Inspect prompts, outputs, errors
- ✅ **Monitoring**: Dashboard for AI performance
- ✅ **Dataset Management**: Store and evaluate test cases
- ✅ **Version Control**: Track prompt changes

**Configuration:**
```env
LANGCHAIN_TRACING_V2=true
LANGCHAIN_API_KEY=lsv2_pt_...
LANGCHAIN_ENDPOINT=https://api.smith.langchain.com
LANGCHAIN_PROJECT=matruraksha
```

### 5. **LangGraph Workflow**
**Purpose:** Orchestrate multi-step AI agents

**Current Use:**
- ASHA chatbot with safety checks
- Doctor AI with structured output

**Future Use:**
- Multi-agent diagnosis system
- Automated referral workflows
- Complex case routing

### 6. **Risk Scoring Engine**
**Algorithm:**
```python
score = 0

# Vital Signs (40 points max)
if bp_systolic > 140: score += 20
if bp_systolic > 160: score += 10 (additional)
if hemoglobin < 11: score += 15
if hemoglobin < 9: score += 10 (additional)
if pulse > 100: score += 5

# Danger Symptoms (40 points max)
if 'bleeding' in symptoms: score += 15
if 'severe_headache' in symptoms: score += 10
if 'decreased_fetal_movement' in symptoms: score += 15
if 'blurred_vision' in symptoms: score += 10
if 'convulsions' in symptoms: score += 20

# Gestational Age (10 points max)
if gestational_age < 28: score += 5  # Preterm risk
if gestational_age > 40: score += 5  # Overdue risk

# Historical Trends (10 points max)
if bp_increasing_trend: score += 5
if weight_loss: score += 5
if hb_declining: score += 5

total_score = min(score, 100)
```

**Risk Categories:**
- 0-30: **LOW** (Green) - Regular monitoring
- 31-60: **MODERATE** (Yellow) - Doctor review recommended
- 61-85: **HIGH** (Orange) - Priority doctor review
- 86-100: **CRITICAL** (Red) - Immediate doctor action
